# MP Analytics — WB & Ozon

Веб-приложение для анализа данных маркетплейсов Wildberries и Ozon. Загружайте Excel-файлы, получайте таблицы аналитики в двух режимах (Краткая сводка / Детальный отчёт) и генерируйте копипаст-промпты для внешнего ИИ.

## Быстрый старт

```bash
# 1. Клонируйте репозиторий
git clone <url>
cd mp-analytics

# 2. Создайте виртуальное окружение
python3 -m venv .venv
source .venv/bin/activate  # macOS/Linux
# .venv\Scripts\activate   # Windows

# 3. Установите зависимости
pip install -r requirements.txt

# 4. Запустите приложение
streamlit run app.py
```

Приложение откроется в браузере по адресу `http://localhost:8501`.

## Какие файлы загружать

Excel-файлы (.xlsx) с данными WB и/или Ozon. Порядок колонок в файле не важен — приложение находит колонки по названиям.

### Обязательные колонки (идентификаторы)

| Колонка | Описание |
|---|---|
| Артикул | Уникальный идентификатор товара |
| Статус | Продается / Новый / Запуск / Выводим |
| Модель | Название модели |
| Склейка на WB | Группа склейки (только для WB) |
| Color code | Код цвета |
| Коллекция | Название коллекции |

### Колонки метрик

Для каждой метрики должны быть два столбца: с суффиксом **п2** (текущий период) и **п1** (прошлый аналогичный период).

Примеры: `Заказы, шт п1`, `Маржа, ₽ п2`, `Продажи до СПП, ₽ п1` и т.д.

Полный список метрик — см. `src/schema.py`.

> **Совет:** заголовки могут содержать лишние пробелы/табы — приложение нормализует их автоматически.

## Вкладки и режимы

| Вкладка | Описание |
|---|---|
| **WB** | Анализ данных Wildberries (краткий / детальный режим) |
| **Ozon** | Анализ данных Ozon (краткий / детальный режим) |
| **Краткая сводка** | Общие KPI; если загружены оба файла — итог WB+Ozon |
| **Промпты для ИИ** | Генерация промптов: краткий (все площадки) или детальный (по площадке) |

### Краткий режим
- Итоговые KPI (п1, п2, дельты)
- ТОП-10 / анти-ТОП-10 артикулов по марже
- Кандидаты на масштабирование
- ТОП моделей
- Воронка WB (если загружены funnel-файлы): KPI воронки + точки роста конверсий

### Детальный режим
Всё из краткого + :
- Реклама и лаг (будущее/настоящее)
- Склад / оборотка с красными флагами
- Лаг цен (ср.чек заказа vs продажи)
- Цветовые аномалии (Color code + Коллекция)
- Склейки WB (только для WB): доноры, паразиты, каннибализация
- Воронка WB: итог, диагностика воронка×экономика, точки роста, агрегации по Модели/Статусу/Склейке
- Агрегации по Модели, Color code+Коллекция, Статусу, Склейке
- Экспорт всех таблиц в Excel (включая funnel-листы)

### Фильтры
В боковой панели доступны фильтры:
- По статусу (Архив выключен по умолчанию)
- По модели
- Поиск по артикулу

### Каноническое определение периодов

| Обозначение | Значение |
|---|---|
| **п2** | Текущий (анализируемый) период |
| **п1** | Прошлый аналогичный период |
| **Δабс** | п2 − п1 (положительное = рост) |
| **Δ%** | (п2 − п1) / \|п1\| × 100. Если п1 = 0 → «—» |

Эта логика единая для всех таблиц, флагов и промптов.

### Настраиваемые пороги (⚙️ в sidebar)

| Параметр | Дефолт | Где используется |
|---|---|---|
| Остатки п2 ≥ (масштабирование) | 200 шт | Кандидаты на масштабирование |
| Оборот п2 ≥ (масштабирование) | 10 дн | Кандидаты на масштабирование |
| DaysCover п2 ≥ | 90 дн | Кандидаты на масштабирование |
| Остатки п2 ≥ (склад) | 100 шт | 🔴 Красный флаг склада |
| Оборот п2 ≥ (склад) | 200 дн | 🔴 Красный флаг склада |
| Δ рекламы ≥ % | 10% | ⚠️ Реклама↑ Заказы↓ |
| Δ рекламы ≥ ₽ | 500 ₽ | ⚠️ Реклама↑ Заказы↓ |
| Порог ratio (аномалии) | 0.3 | Цветовые аномалии |
| Мин. макс. маржа (аномалии) | 1000 ₽ | Цветовые аномалии |

## Воронка WB (Funnel)

В боковой панели можно загрузить **2 дополнительных Excel-файла** с воронкой продаж WB:
- **WB Funnel п1** — прошлый период
- **WB Funnel п2** — текущий период

### Формат файлов воронки
- В книге должен быть лист **«Товары»** (строго).
- Ключевая колонка: `Артикул продавца` — соответствует `Артикул` из экономики.
- Колонки: Показы, Переходы в карточку, Положили в корзину, Заказали шт, Выкупили шт, Отменили шт, суммы ₽ и др.

### Что рассчитывается
- **CTR, %** = Переходы / Показы × 100
- **Conv2Cart, %** = Корзина / Переходы × 100
- **Conv2Order, %** = Заказали / Переходы × 100
- **PurchaseRate, %** = Выкупили / Заказали × 100

Конверсии пересчитываются из агрегированных абсолютных показателей (не берутся из исходных процентов).

### Связь с экономикой
- Join по нормализованному артикулу (trim, lower, ё→е, пробелы вокруг дефисов).
- Показывается coverage: сколько артикулов из экономики сопоставлено с воронкой.
- Диагностика: Реклама↑Трафик↓, Цена↑Конверсия↓, падение показов.
- Точки роста: высокий трафик + низкая конверсия → потенциал улучшения.

## WB API (Funnel + Ads)

Приложение поддерживает получение данных воронки и рекламы WB напрямую через API.

### Настройка токена

1. Создайте файл `.streamlit/secrets.toml` (он в `.gitignore`):
   ```toml
   [wb]
   token = "ваш_токен_WB_API"
   token2 = "второй_токен_WB_API"  # опционально
   ```
2. Или задайте переменные окружения `WB_TOKEN` и `WB_TOKEN_2`.

Оба токена используются одновременно: данные воронки и рекламы загружаются по каждому ключу и объединяются с дедупликацией по `nmId`.

**Токены никогда не логируются и не коммитятся** — проверяйте `.gitignore`.

### Функции API

- **Funnel API v3** — `/api/analytics/v3/sales-funnel/products` — воронка продаж по карточкам (альтернатива Excel).
- **Ads API v3** — `/adv/v3/fullstats` — статистика рекламных кампаний (true ads: показы, клики, CTR, CPC, CR, spend).
- **API Inspector** — вкладка для тестовых вызовов, просмотра JSON, schema summary, сохранения samples.

### Matching nmId → Артикул

1. **Через воронку**: nmId → vendorCode (из API) → нормализация → Артикул (экономика).
2. **Через override**: ручное сопоставление nmId → Артикул (JSON файл или UI).
3. Coverage: показывается % matched spend.

### Ads-метрики (True Ads)

| Метрика | Формула |
|---|---|
| True CTR (ads) | Клики / Показы × 100 |
| CPC | Затраты / Клики |
| CPM | Затраты / Показы × 1000 |
| CR (ads) | Заказы / Клики × 100 |
| CartRate (ads) | Корзина / Клики × 100 |

> **ВАЖНО**: Funnel CTR ≠ True Ads CTR. Funnel = весь трафик, Ads = только рекламный.

## Вычисляемые метрики

- **ROI, %** = (Маржа, ₽ / Себес-ть, ₽) × 100
- **Годовая доходность, %** = ROI / Оборот продаж (дни) × 365
- **Δабс** = п2 − п1
- **Δ%** = (п2 − п1) / |п1| × 100
